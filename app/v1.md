# 项目技术文档

## 项目概览
- 目标：通过自然语言的模糊描述，检索音频节目中对应内容的时刻，支持“有趣片段”发现（笑声加权）。
- 架构：离线解析与索引 → 语义检索服务 → Web UI 搜索与播放。
- 约束：运行于 MacBook Air，尽量使用轻量模型与 CPU 友好实现。

## 目录结构
- `app/web.py`：后端入口与 HTTP API
- `app/audio_processing.py`：音频转写、说话人归属、笑声检测、索引构建
- `app/search_service.py`：索引加载与自然语言语义检索
- `app/static/index.html`、`app/static/app.js`：Web UI
- `app/split_audio.py`：音频切分工具
- `app/uploads/`：上传音频与切分产物
- `app/data/`：解析后的索引与嵌入

示例：
- 切分产物：`app/uploads/chunks/<base>/chunk_000.wav`
- 索引结果：`app/data/<audio_basename>/index.json`、`embeddings.npy`

## 运行环境与依赖
- Python 3.9（系统环境），或使用虚拟环境
- 主要依赖：
  - `flask`（Web 服务）
  - `openai-whisper`（转写，默认 `tiny`，CPU 友好）
  - `librosa`、`soundfile`（音频 I/O 与特征）
  - `sentence-transformers`、`transformers`（语义嵌入；失败时回退 TF‑IDF）
  - `resemblyzer`、`webrtcvad`（说话人嵌入与语音活动）

安装示例（用户环境）：
```
pip3 install --user flask openai-whisper librosa soundfile sentence-transformers transformers resemblyzer webrtcvad scipy numpy
```

## 核心模块

**音频转写与时间戳**
- `transcribe_audio` 在 `app/audio_processing.py:6`
- 加载波形：`librosa.load(..., sr=16000, mono=True)`，避免 `ffmpeg` 依赖
- Whisper 调用：`model.transcribe(y, fp16=False)`，返回分段文本与起止时间

**说话人归属（粗粒度）**
- `assign_speakers` 在 `app/audio_processing.py:21`
- 用 `resemblyzer.VoiceEncoder` 对片段嵌入，余弦相似度与在线原型更新进行聚类，得到 speaker id

**笑声检测（启发式）**
- `detect_laughter` 在 `app/audio_processing.py:49`
- 文本关键词（“哈哈”“笑”等）与声学特征（零交叉率、RMS 能量）合成分值，值域约 `[0, 1]`
- 调用修正：`librosa.feature.zero_crossing_rate(y=clip)` 与 `librosa.feature.rms(y=clip)`（位置 `app/audio_processing.py:60–63`）

**索引构建与保存**
- `build_index` 在 `app/audio_processing.py:74`
- 生成条目数组与文本嵌入：
  - 优先 `SentenceTransformer("distiluse-base-multilingual-cased-v2")`
  - 出错时回退 TF‑IDF，保存 `tfidf.pkl`
- 保存：
  - 嵌入：`embeddings.npy`
  - 索引：`index.json`，格式为 `{"items": [...], "method": "st|tfidf"}`

**检索服务**
- `SearchService` 在 `app/search_service.py:5`
- 加载索引与嵌入，识别嵌入方法（`st` 或 `tfidf`）：
  - `index.json` 解析于 `app/search_service.py:15–23`
  - 回退模型加载于 `app/search_service.py:24–38`
- 检索：
  - 查询嵌入（ST 或 TF‑IDF）
  - 相似度打分：余弦相似度 + 笑声加权（`score = sim + 0.2 * laughter`），实现位置 `app/search_service.py:44–52`

## 数据流程
1. 上传音频或指定本地文件
2. 转写为文本段，获取时间戳
3. 说话人归属（粗分类）
4. 笑声检测并生成分值
5. 文本向量化（ST 或 TF‑IDF）
6. 索引落盘（JSON + NPY）
7. 查询时将自然语言转为向量，计算相似度并加权排序，返回片段与时间

## API 设计

- `GET /`：静态页面 `index.html`
- `GET /media/<path>`：静态音频访问（相对路径基于 `uploads`）
- `POST /api/index`：上传文件并解析，返回索引信息
  - 入口位置 `app/web.py:46`
- `GET /api/test_index?name=<path>&timeout=<sec>`：解析本地上传文件（支持子目录路径），可设置超时
  - 支持相对路径解析，安全限定于 `uploads`，位置 `app/web.py:58–86`
- `GET /api/test_index_async?name=<path>`：异步触发解析，返回 `task_id`
  - 位置 `app/web.py:88–106`
- `GET /api/task_status?id=<task_id>`：查询任务状态与结果
  - 位置 `app/web.py:108–118`
- `GET /api/split?name=<path>&sec=<秒>&overlap=<秒>`：将音频切分为固定长度 `wav`
  - 位置 `app/web.py:120–146`
  - 工具函数 `split_audio` 在 `app/split_audio.py:1`
- `GET /api/search?q=<query>&id=<index_id>`：语义检索，返回片段列表
  - 位置 `app/web.py:148–160`

任务管理：
- 异步任务管理于 `TASKS`（内存），线程执行与状态在 `app/web.py:11–29`

## 前端 UI
- `app/static/index.html`：上传文件、搜索输入框、结果列表、播放器
- `app/static/app.js`：
  - 上传调用 `/api/index`，保存 `index id` 与 `audio_url`
  - 搜索调用 `/api/search?q=...&id=...`
  - 播放跳转到结果片段起始时间

## 索引格式
- 文件：`app/data/<basename>/index.json`
- 内容示例：
```
{
  "method": "st",
  "items": [
    {
      "i": 0,
      "text": "...",
      "start": 0.0,
      "end": 0.92,
      "speaker": 0,
      "laughter": 0.5,
      "audio_path": "/abs/path/to/file.wav"
    }
  ]
}
```
- 嵌入：`embeddings.npy`
- TF‑IDF 回退时另存：`tfidf.pkl`

## 检索算法
- 查询嵌入：
  - ST 模式：`distiluse-base-multilingual-cased-v2`
  - TF‑IDF 模式：`TfidfVectorizer(max_features=2048)`
- 相似度与加权：
  - 计算 `sims = emb_matrix @ query_emb`
  - 排序分数 `score = sims[i] + 0.2 * laughter[i]`
- 实现参考：
  - 加权公式 `app/search_service.py:44–52`

## 性能与资源
- Whisper 模型：默认 `tiny`，CPU 可用；较长音频建议异步解析或切分处理
- 估算（以 100MB MP3）：
  - 128 kbps ≈ 1.8 小时音频；CPU 实时倍率 0.5×–2×，处理约 1–4 小时
- 切分建议：`sec=600`（10 分钟）分块并行或队列处理

## 常见问题
- `ffmpeg` 缺失：已通过 `librosa.load` 直接波形输入，避免依赖
- `sentence-transformers` 下载/SSL 问题：自动回退 TF‑IDF，保障索引构建
- `librosa.feature.rms` 参数错误：使用关键字参数 `y=clip`（修复于 `app/audio_processing.py:60–63`）
- 路径问题：`name` 支持相对子路径，使用 `_resolve_upload_path` 做安全约束（`app/web.py:12`）
